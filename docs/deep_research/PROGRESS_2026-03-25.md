## 三、已解决的问题 (Debug Log)

| # | 问题 | 原因 | 解决方案 |
|---|------|------|----------|
| 1 | `'list' object has no attribute 'get'` | LLM 返回 `[{...}]` 而非 `{"analysts": [...]}` | 使用 `with_structured_output(Perspectives)` 强制格式 |
| 2 | `KeyError: 'affiliation'` | LLM 返回错误字段名 (如 `expertise`, `focus`) | 更新 prompt 明确要求 `name, affiliation, role, description` |
| 3 | `AttributeError: 'str' object has no attribute 'get'` | `JsonOutputParser` 返回字符串非字典 | 改用 `with_structured_output(SearchQuery)` |
| 4 | `JSONDecodeError` | LLM 输出不是合法 JSON | `with_structured_output` 内部处理解析 |
| 5 | `ValueError: Got unsupported message type: ('user', ...)` | `get_buffer_string()` 需要 `BaseMessage` 对象 | 使用 `HumanMessage(content=...)` 而非元组 |
| 6 | `Deserializing unregistered type Analyst` | `MemorySaver` 无法序列化 Pydantic 对象 | 使用 `.model_dump()` 转为字典 |
| 7 | `LangGraphDeprecatedSinceV10` | `Send` 从 `langgraph.constants` 导入 | 改为从 `langgraph.types` 导入 |
| 8 | `OPENAI_API_KEY not set` | dotenv 未加载 | 确保 `.env` 存在且 `main.py` 加载它 |
| 9 | 事件结构检查错误 | `astream_events` 格式是 `{"event":..., "name":...}` | 使用 `event.get("name", "")` |

## 四、已解决的问题

### Issue #1: Graph 执行和 API 返回问题 ✅ 已解决

**原因:**
1. `DASHSCOPE_API_KEY=sk-a6c43cb` 在 worktree 的 `.env` 中被截断
2. `route_messages` 函数检查 `messages[-1]` 而非 `messages[-2]` 导致无法正确结束访谈
3. `search_web/bocha` 中 `search_instructions` 没有被正确包装为 `SystemMessage`
4. `interrupt_before=["human_feedback"]` 在无反馈场景下导致 graph 无法继续

**修复:**
1. 补全 `DASHSCOPE_API_KEY` 完整密钥
2. 修复 `route_messages` 检查 `messages[-2]`
3. 所有节点函数添加 try-except 错误处理
4. 移除 `interrupt_before` 实现同步执行

**验证结果 (2026-03-26):**
- 直接调用 `graph.invoke()` → 成功，final_report: 1136 chars, sections: 2
- API 调用 `/execute` → 成功，final_report: 2243 chars, sections: 2
- 数据库正确保存报告
