## 🏆 技术亮点

### 1. 国产AI模型应用 🇨🇳

**DeepSeek**：
- 模型：deepseek-chat
- 优势：成本低（约GPT-4的1/10）、性能好、中文支持优秀
- 实现：完整的异步封装、流式输出、重试机制

**通义千问**：
- 模型：text-embedding-v3
- 优势：中文效果好、维度适中（1024维）、价格合理
- 实现：批量向量化、异步处理

**成本优势**：
- 单用户月度成本：约¥2（vs GPT-4的¥10+）
- 1000用户规模：约¥2000/月（vs GPT-4的¥10000+）
- **成本降低约80%！**

### 2. 完整的RAG架构

```
用户问题
    ↓
向量化（通义千问）
    ↓
语义检索（Milvus）
    ↓
上下文构建
    ↓
Prompt注入（用户画像+检索结果）
    ↓
答案生成（DeepSeek）
    ↓
来源标注 + 置信度
    ↓
返回给用户
```

### 3. 分层记忆架构

```
短期记忆（Redis）
- 会话内对话
- 2小时过期
- < 10ms访问
    ↓
工作记忆（Redis + PostgreSQL）
- 最近7天活动
- 快速访问
- < 50ms
    ↓
长期记忆（PostgreSQL + Milvus）
- 用户画像
- 重要对话
- 永久保留
- < 200ms
```

### 4. 异步编程设计

- 全面使用 async/await
- 异步数据库操作
- 异步HTTP请求
- 并发处理能力强

### 5. 模块化架构

```
API层（endpoints）
    ↓
服务层（services）- 业务逻辑
    ↓
AI层（ai/）- RAG、记忆、Agent
    ↓
数据层（models/db）- 持久化
```

清晰的分层，易于维护和扩展。

## 🔍 功能演示

### 演示1：完整的对话流程

```python
import requests

BASE_URL = "http://localhost:8000/api/v1"

# 1. 注册用户
response = requests.post(f"{BASE_URL}/auth/register", json={
    "username": "demo_user",
    "email": "demo@example.com",
    "password": "demo123456"
})

# 2. 登录
response = requests.post(f"{BASE_URL}/auth/login", json={
    "username": "demo_user",
    "password": "demo123456"
})
token = response.json()["access_token"]
headers = {"Authorization": f"Bearer {token}"}

# 3. 设置画像
requests.put(f"{BASE_URL}/profile", headers=headers, json={
    "name": "小张",
    "occupation": "产品经理",
    "learning_goal": "project"
})

# 4. 生成学习路径
response = requests.post(f"{BASE_URL}/learning-path/generate", 
    headers=headers,
    json={
        "learning_goal": "project",
        "available_hours_per_week": 10
    }
)
print("学习路径：", response.json())

# 5. 开始学习对话
response = requests.post(f"{BASE_URL}/chat", headers=headers, json={
    "message": "我想学习RAG系统，从哪里开始？"
})
print("回答：", response.json()["message"])
print("来源：", response.json()["sources"])

# 6. 查看进度
response = requests.get(f"{BASE_URL}/progress/stats", headers=headers)
print("进度统计：", response.json())
```

```bash

```

**Q2: API返回500错误？**
- 查看应用日志
- 检查数据库连接
- 检查API密钥配置

**Q3: 向量化失败？**
- 检查DASHSCOPE_API_KEY
- 检查网络连接
- 查看日志详细错误

**Q4: 对话没有返回内容？**
- 确保已导入知识库
- 检查DEEPSEEK_API_KEY
- 查看检索是否有结果

