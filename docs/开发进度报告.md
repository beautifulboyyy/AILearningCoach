## 三、技术实现细节

### 3.1 技术栈

#### 后端框架
- FastAPI 0.109+（异步Web框架）
- Uvicorn（ASGI服务器）
- SQLAlchemy 2.0+（异步ORM）
- Alembic（数据库迁移）

#### AI技术
- DeepSeek API（deepseek-chat模型）
- 通义千问 DashScope API（text-embedding-v3）
- LangChain（AI框架）
- Milvus 2.3+（向量数据库）

#### 数据存储
- PostgreSQL 15（关系数据库）
- Redis 7（缓存）
- Milvus（向量存储）

#### 其他工具
- Pydantic 2.0+（数据验证）
- Loguru（日志）
- Python-Jose（JWT）
- Passlib（密码哈希）
- Tenacity（重试机制）

### 3.2 核心技术实现

#### RAG系统架构
```
用户问题
    ↓
向量化（通义千问Embedding）
    ↓
Milvus向量检索（Top-K）
    ↓
上下文格式化
    ↓
Prompt构建（注入用户画像）
    ↓
DeepSeek生成答案
    ↓
返回答案+来源+置信度
```

#### 数据流设计
1. **用户认证流程**：
   - 用户登录 → JWT生成 → Token验证 → 权限检查

2. **对话流程**：
   - 接收消息 → 会话管理 → RAG检索 → LLM生成 → 保存历史

3. **向量检索流程**：
   - 查询向量化 → Milvus搜索 → 相似度过滤 → 结果排序

### 3.3 性能优化

- ✅ 异步I/O（FastAPI + AsyncPG）
- ✅ 连接池（数据库连接池）
- ✅ 缓存机制（Redis缓存）
- ✅ 批量向量化（提高效率）
- ✅ 错误重试（提高稳定性）
