# AI学习教练系统 - 项目交付说明

## 🎉 项目概述

本项目是一个基于**DeepSeek**和**通义千问**的智能学习教练系统，为LLM应用开发课程的学习者提供个性化学习支持。

**核心特性**：
- 🤖 智能问答（基于RAG技术）
- 🧠 长期记忆（跨会话记住用户）
- 📚 个性化学习路径
- 📊 学习进度追踪
- ✅ 任务管理

---

## ✅ 已完成的核心功能

### 1. 用户认证系统
- 用户注册、登录
- JWT Token认证
- Token刷新机制
- 权限验证

### 2. RAG智能问答系统
- 使用**通义千问 text-embedding-v3**向量化
- 使用**Milvus**向量数据库存储和检索
- 使用**DeepSeek deepseek-chat**生成答案
- 支持流式和非流式输出
- 自动标注知识来源和置信度

### 3. 记忆管理系统
- 用户画像管理（背景、目标、偏好）
- 短期记忆（会话内）
- 工作记忆（7天）
- 长期记忆（永久）
- 记忆查询、删除、导出

### 4. 学习规划系统
- 个性化学习路径生成
- 基于用户背景和目标
- 分阶段学习计划
- 路径动态调整

### 5. 任务管理系统
- 任务CRUD
- 优先级和截止日期
- 任务完成跟踪
- 状态管理

### 6. 学习进度追踪
- 模块进度记录
- 完成度统计
- 学习报告生成
- 周报、月报

## 🚀 如何使用

### 前置要求

1. **API密钥**
   - DeepSeek API Key（https://platform.deepseek.com/）
   - 通义千问 API Key（https://dashscope.console.aliyun.com/）

2. **Docker环境**
   - Docker Desktop已安装

### 启动步骤

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑.env，填入API密钥

# 2. 启动服务
docker-compose up -d

# 3. 初始化数据库
docker-compose exec app python scripts/init_db.py

# 4. 初始化Milvus
docker-compose exec app python scripts/init_milvus.py

# 5. 创建测试用户
docker-compose exec app python scripts/create_test_user.py

# 6. 访问API文档
open http://localhost:8000/docs
```

### 测试接口

```bash
# 登录
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "testuser", "password": "test123456"}'

# 对话（替换YOUR_TOKEN）
curl -X POST http://localhost:8000/api/v1/chat \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"message": "你好，我想学习AI"}'
```

## 📖 API文档

### 核心API

#### 认证（/api/v1/auth）
- `POST /register` - 注册
- `POST /login` - 登录
- `GET /me` - 获取用户信息

#### 对话（/api/v1/chat）
- `POST /` - 发送消息
- `POST /stream` - 流式对话
- `GET /history/{session_id}` - 对话历史

#### 用户画像（/api/v1/profile）
- `GET /` - 获取画像
- `PUT /` - 更新画像
- `POST /generate` - 自动生成

#### 学习路径（/api/v1/learning-path）
- `POST /generate` - 生成路径
- `GET /active` - 获取活跃路径
- `PUT /{id}` - 更新路径

#### 任务（/api/v1/tasks）
- `POST /` - 创建任务
- `GET /` - 任务列表
- `POST /{id}/complete` - 完成任务

#### 进度（/api/v1/progress）
- `GET /stats` - 进度统计
- `GET /report/weekly` - 周报

## 💡 使用示例

### 场景1：新用户注册并开始学习

```python
import requests

# 1. 注册
response = requests.post("http://localhost:8000/api/v1/auth/register", json={
    "username": "student01",
    "email": "student01@example.com",
    "password": "pass123456"
})

# 2. 登录
response = requests.post("http://localhost:8000/api/v1/auth/login", json={
    "username": "student01",
    "password": "pass123456"
})
token = response.json()["access_token"]

headers = {"Authorization": f"Bearer {token}"}

# 3. 更新画像
requests.put("http://localhost:8000/api/v1/profile", headers=headers, json={
    "name": "小王",
    "occupation": "软件工程师",
    "learning_goal": "job_hunting"
})

# 4. 生成学习路径
response = requests.post("http://localhost:8000/api/v1/learning-path/generate", 
    headers=headers, 
    json={
        "learning_goal": "job_hunting",
        "available_hours_per_week": 10
    }
)
print(response.json())

# 5. 开始提问
response = requests.post("http://localhost:8000/api/v1/chat", 
    headers=headers,
    json={"message": "什么是Prompt工程？"}
)
print(response.json())
```

### 场景2：查看学习进度

```python
# 获取进度统计
response = requests.get("http://localhost:8000/api/v1/progress/stats", headers=headers)
print(response.json())

# 获取周报
response = requests.get("http://localhost:8000/api/v1/progress/report/weekly", headers=headers)
print(response.json())
```

### 常见问题排查

2. **API 401错误**：检查token是否有效
3. **对话无响应**：检查API密钥、知识库是否导入

```bash

docker-compose restart app

# 重启所有服务
```

### 1. 知识库为空
