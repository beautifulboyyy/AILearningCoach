## 🚀 快速启动

### 1. 安装Node.js

确保已安装 Node.js 16+ 版本：

```bash
node --version  # 应该显示 v16.x.x 或更高
```

### 2. 进入前端目录

```bash
cd /Users/zhulang/work/AILearningCoach/frontend
```

### 3. 安装依赖

```bash
npm install
```

> **注意**: 首次安装可能需要5-10分钟，请耐心等待。

### 4. 确保后端服务运行

在启动前端之前，确保后端服务已经启动：

```bash
# 在另一个终端窗口
cd /Users/zhulang/work/AILearningCoach
# 启动后端服务（Docker方式）
docker-compose up -d
```

验证后端是否运行：
- 访问 http://127.0.0.1:8000/docs 查看API文档
- 确保数据库和Redis服务正常

### 5. 启动开发服务器

```bash
npm run dev
```

成功启动后会看到：

```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
  ➜  press h to show help
```

### 6. 访问应用

打开浏览器访问：http://localhost:3000

## 🔧 常见问题

### 1. 依赖安装失败

如果 `npm install` 失败，尝试：

```bash
# 清除缓存
npm cache clean --force

# 删除node_modules和lock文件
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 2. 启动失败

检查端口3000是否被占用：

```bash
# macOS/Linux
lsof -ti:3000 | xargs kill -9

# 或修改端口
# 编辑 vite.config.ts 中的 server.port
```

### 3. 无法连接后端API

确认：
- 后端服务运行在 http://127.0.0.1:8000
- `.env.development` 中的 API地址正确
- 检查浏览器控制台网络请求

### 4. 页面空白

打开浏览器开发者工具（F12）查看：
- Console面板是否有错误
- Network面板API请求是否成功

## 📦 生产环境构建

### 构建项目

```bash
npm run build
```

构建产物在 `dist/` 目录中。

### 预览构建结果

```bash
npm run preview
```

### 部署到服务器

将 `dist/` 目录的内容上传到Web服务器（如Nginx）：

```nginx
# Nginx配置示例
server {
    listen 80;
    server_name your-domain.com;
    
    root /path/to/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://127.0.0.1:8000;
    }
}
```
