## 面试 90 秒项目介绍模板
我做的是一个 AI 学习教练系统，技术栈是 FastAPI + Vue3 + Milvus + Celery，部署在 Docker Compose。核心亮点是把聊天做成了流式 SSE，并在后端做了 Multi-Agent 编排：先识别用户意图，再路由到不同 Agent。问答场景下会走 RAG，先检索再生成，减少幻觉。系统还把对话和学习进度关联起来，通过 Celery 异步更新用户画像和进度，保证主链路响应速度。这个项目让我从“会调接口”升级到“能讲清架构、定位问题并做优化”。

## 高频追问准备
1. 为什么用 SSE？
答：实现简单，适配文本流式场景，前端感知延迟更低。

2. 为什么需要 Celery？
答：把画像更新、进度同步等非核心耗时任务从主请求剥离，降低接口时延。

3. Multi-Agent 相比单模型有什么收益？
答：职责清晰，可独立优化，错误定位更快，后续扩展新 Agent 成本低。

4. 你做了哪些真实工作？
答：部署、链路梳理、流式交互理解、异步任务理解、故障排查与复盘。
