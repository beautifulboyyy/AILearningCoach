## 二、开发阶段规划

### 阶段划分原则

1. **按功能优先级**：P0 → P1 → P2
2. **按依赖关系**：基础设施 → 核心功能 → 增强功能
3. **按可测试性**：每个阶段交付可独立测试的功能模块
4. **迭代式开发**：每周一个迭代，快速验证

### 总体时间线

| 阶段 | 周期 | 核心目标 | 可交付成果 |
|-----|------|---------|-----------|
| **第1周** | 基础设施完善 | 数据库设计、项目结构、开发环境 | 完整的数据库表结构、开发文档 |
| **第2周** | RAG系统MVP | 知识库构建、基础问答 | 可用的技术问答API |
| **第3周** | 记忆系统基础版 | 用户画像、短期记忆 | 用户信息记忆功能 |
| **第4周** | 记忆系统完整版 | 长期记忆、跨会话 | 完整的记忆管理系统 |
| **第5周** | 学习规划系统 | 个性化路径生成 | 学习规划API |
| **第6周** | 任务与进度管理 | 任务CRUD、进度追踪 | 任务管理API |
| **第7周** | Multi-Agent系统 | Agent编排、协作 | 智能路由和Agent协作 |
| **第8周** | 优化与测试 | 性能优化、测试完善 | 生产就绪的后端系统 |

## 三、第1周：基础设施完善

### 3.1 数据库设计（1-2天）

**核心表结构设计**：

1. **users（用户表）**
   - id, username, email, password_hash
   - created_at, updated_at, last_login

2. **user_profiles（用户画像表）**
   - id, user_id, name, age, occupation
   - technical_background (JSON)
   - learning_goal, learning_preference
   - current_level (JSON)
   - created_at, updated_at

3. **conversations（会话表）**
   - id, user_id, session_id
   - started_at, ended_at, message_count

4. **messages（消息表）**
   - id, conversation_id, user_id
   - role (user/assistant), content
   - metadata (JSON: sources, memory_used)
   - created_at

5. **memories（记忆表）**
   - id, user_id, memory_type (short/working/long)
   - content, content_vector
   - importance_score, created_at, last_accessed
   - expires_at

6. **learning_paths（学习路径表）**
   - id, user_id, title, description
   - phases (JSON), status
   - created_at, updated_at

7. **tasks（任务表）**
   - id, user_id, learning_path_id
   - title, description, priority
   - status, due_date
   - created_at, updated_at, completed_at

8. **learning_progress（学习进度表）**
   - id, user_id, module_name
   - status, completion_percentage
   - started_at, completed_at

9. **knowledge_chunks（知识块表）**
   - id, lecture_number, section
   - content, content_vector
   - metadata (JSON), difficulty_level
   - created_at

## 十一、关键技术实现方案

### 11.1 RAG系统技术方案

**知识库构建**：
```
1. 文档预处理
   - Markdown解析
   - 代码块识别
   - 章节分割

2. 智能分块
   - Chunk大小: 500-800 tokens
   - 重叠: 100 tokens
   - 保持代码块完整

3. 向量化
   - 模型: BGE-large-zh
   - 存储: Milvus
   - 索引: IVF_FLAT
```

**检索策略**：
```
1. 向量检索
   - Top-K: 5-10
   - 相似度阈值: 0.7

2. 混合检索
   - 向量检索 + BM25
   - 权重: 0.7:0.3

3. Reranking
   - 使用BGE-reranker
   - 最终返回Top-3
```

### 11.2 记忆系统技术方案

**记忆压缩**：
```
传统方式: 完整对话历史 (5000+ tokens)
优化方式: 结构化摘要 (300-500 tokens)

压缩策略:
1. 提取关键信息 (LLM)
2. 结构化存储
3. 按需召回
4. 动态控制召回量
```

**检索优化**：
```
三级检索:
1. 工作记忆 (Redis, <10ms)
2. 画像查询 (PostgreSQL, <50ms)
3. 语义检索 (Milvus, <200ms)

时间衰减:
score = similarity * (1 / (1 + age_days))
```

### 11.3 Multi-Agent技术方案

**架构设计**：
```
使用LangGraph实现:
1. 定义Agent节点
2. 定义路由逻辑
3. 定义状态管理
4. 定义结束条件

意图识别:
- 使用小模型快速分类
- 置信度 > 0.8 直接路由
- 置信度 < 0.8 使用默认QA Agent
```
