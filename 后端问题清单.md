## ⚠️ 接口设计问题（需要确认）

### 问题3: Intent接口参数位置不明确 ⚠️ **P1**

**端点**: `POST /api/v1/agents/intent`  
**状态码**: 422  
**错误信息**: Field required in 'query', 'message'

**问题描述**:
- 测试使用`json`传参：`json={"message": "..."}`
- 接口期望`query`参数：`params={"message": "..."}`
- 参数位置不清晰

**影响范围**:
- API使用者容易混淆
- 文档可能不够清晰

**当前接口定义**:
```python
# app/api/v1/endpoints/agents.py
@router.post("/intent")
async def test_intent(
    message: str = Query(...),  # ← Query参数，不是Body
    current_user: User = Depends(get_current_active_user)
):
```

**问题分析**:
- POST请求通常使用Request Body
- 但此接口使用Query参数（不符合REST惯例）

**修复优先级**: P1（中优先级）

**建议方案**:

**方案A: 改为Body参数（推荐）** ✅
```python
from pydantic import BaseModel

class IntentRequest(BaseModel):
    message: str

@router.post("/intent")
async def test_intent(
    request: IntentRequest,  # ← Body参数
    current_user: User = Depends(get_current_active_user)
):
    message = request.message
```

**方案B: 保持Query参数，更新文档** 📝
```python
# 明确说明使用Query参数
# 在Swagger文档中注明
```

**推荐**: 使用方案A，符合REST规范

**问题**:
- POST请求通常不使用Query参数
- Query参数应该用于GET请求的过滤/排序

**问题**: JWT的`sub`字段必须是字符串，但传入了整数

**修复**:
```python
# app/core/security.py
def create_access_token(data: dict, ...) -> str:

        to_encode["sub"] = str(to_encode["sub"])  # ✅ 转换为字符串
```

**效果**: 认证系统完全正常 ✅
