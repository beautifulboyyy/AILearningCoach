### 问题1: Intent接口参数设计 ✅ **P1**

**问题描述**: 
- POST请求使用Query参数而非Body参数
- 不符合REST规范

**修复内容**:
```python
# 修复前
@router.post("/intent")
async def identify_intent(
    message: str,  # Query参数
    ...
):

# 修复后
class IntentRequest(BaseModel):
    message: str
    context: Optional[Dict] = None

@router.post("/intent")
async def identify_intent(
    request: IntentRequest,  # Body参数
    ...
):
```

### 问题4: 学习路径生成超时 ✅ **P0**

**问题描述**:
- LLM生成耗时>30秒，导致HTTP超时
- 用户体验差

**1. 简化Prompt**（Token从2000减到800，-60%）
```python

planning_prompt = f"""{LEARNING_PLANNER_SYSTEM_PROMPT}

{profile_info}

## 学习需求
- 学习目标：{learning_goal}
- 每周可用时间：{available_hours} 小时
...（大量详细说明）

请以JSON格式返回，格式如下：
{{
    "title": "学习路径标题",
    "description": "整体描述",
    ...（详细字段说明）
}}
"""

# 修复后（精简版，800 tokens）
planning_prompt = f"""生成学习路径（JSON格式，简洁回答）：

目标：{learning_goal}
每周时间：{available_hours}小时

模块：1.Prompt工程 2.Function Calling 3.RAG系统 4.Agent开发 5.MCP

返回JSON（不超过500字）：
{{
    "title": "路径标题",
    ...
}}

只返回JSON。"""
```

**2. 优化LLM参数**
```python
# 修复前

)

# 修复后
response = await llm.generate(
    messages=[{"role": "user", "content": planning_prompt}],
    temperature=0.5,  # 降低随机性，加快生成
    max_tokens=800    # 限制回复长度
```
